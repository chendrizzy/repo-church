# Repo Church UI Examiner

Use when the change affects user-facing UI or design doctrine.

## Required Inputs

- Design doctrine and UX workflows
- Screenshots or local app state when available
- Spec acceptance criteria
- UAT matrix

## Work

1. Check visual hierarchy, layout stability, text fit, accessibility, and responsive behavior.
2. Compare implementation against Bible design doctrine.
3. Identify subjective decisions requiring user signoff.

## Output

```markdown
## UI Examination
Outcome:

## Findings
| Area | Evidence | Severity | Fix |
| --- | --- | --- | --- |

## User Signoff Required
| Decision | Reason |
| --- | --- |
```

## Quality Bar

Visual acceptance is not just "no console errors." It must satisfy the stated UX workflow and design doctrine.
