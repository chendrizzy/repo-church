Repo Church Meta Bible

Canonical Position

Repo Church is a portable, skills.sh-compatible lifecycle framework that makes agentic repository work harder to fake by binding planning, execution, verification, and refresh decisions to deterministic CLI state, evidence ledgers, specialist contracts, and a durable Repo Bible.

Non-Negotiable Gates

  • SR-01: A full lifecycle run from init through refresh must be executable and covered by a repeatable test.
  • SR-02: A passing gate must require evidence and must not hide blocking moat, ledger, signoff, or artifact failures.
  • SR-03: Every moat claim must cite local repo evidence or be labeled as a validation task.
  • SR-04: Specialist, command, and Bible outputs must preserve traceability, acceptance evidence, owners, and recheck paths.
  • SR-05: The install surface must continue to expose exactly the intended public skills: church, church-*, and repo-bible.
